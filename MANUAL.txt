SYNAPSEQ(1)
SynapSeq Manual development

NAME
    synapseq

    text-driven audio sequencer for brainwave sessions and ambient sound
    design

SYNOPSIS
        synapseq [OPTION]... INPUT [OUTPUT]
        synapseq -new TYPE
        synapseq -preview INPUT
        synapseq -play INPUT
        synapseq -mp3 INPUT
        synapseq -test INPUT
        synapseq -hub-update
        synapseq -hub-clean
        synapseq -hub-list
        synapseq -hub-search WORD
        synapseq -hub-info NAME
        synapseq -hub-download NAME [DIR]
        synapseq -hub-get NAME [OUTPUT]
        synapseq -manual
        synapseq -help
        synapseq -version
        synapseq -manual -no-color

    Paged reading

    Linux and macOS
        synapseq -manual | less

    Windows PowerShell
        synapseq -manual | more

DESCRIPTION
    SynapSeq parses a line-oriented score language and renders deterministic
    audio. Ordering is strict. Sequence options must appear first. Presets
    must appear next. Timeline entries must appear last.

    INPUT may be a local .spsq file, a sequence URL, or -. Output is WAV by
    default. -mp3 selects MP3. -preview selects HTML. -play selects direct
    playback. -test validates only.

SOUND CONCEPTS
    binaural
        A pair of nearby tones, one per ear, that creates a perceived beat
        inside the listener. Headphones are recommended because each ear must
        receive a different carrier.

    monaural
        A beat created by mixing the tones before playback. The pulse is
        present in the audio signal itself, so it can be heard on speakers or
        headphones.

    isochronic
        A single carrier that is gated on and off at the beat rate. The result
        is a clearly pulsed rhythm with sharp entrainment cues.

    doppler
        An effect for tone tracks that shifts motion and pitch perspective so
        the sound feels like it is moving toward and away from the listener.

    pan
        An effect that moves energy between the left and right channels. Use
        it to widen static sounds or create gentle spatial motion.

    modulation
        An effect that varies amplitude over time. Use it to add movement,
        shimmer, or slow breathing to a track.

OPTIONS
    Rules

        one primary action per command
        INPUT required except when not applicable
        OUTPUT optional
        derived output path when OUTPUT is omitted

    General

    -new TYPE
        create a starter sequence

        Example

            synapseq -new meditation

    TYPE
        meditation
        focus
        sleep
        relaxation
        example

    -test
        validate syntax and semantics only

        Example

            synapseq -test session.spsq

    -preview
        render HTML preview

        Example

            synapseq -preview session.spsq

    -play
        render and play through ffplay

        Example

            synapseq -play session.spsq

    -mp3
        render MP3 output

        Example

            synapseq -mp3 session.spsq

    -quiet
        suppress non-error CLI output

        Example

            synapseq -quiet session.spsq

    -no-color
        disable ANSI color output

        Example

            synapseq -manual -no-color

    -manual
        print full manual

        Example

            synapseq -manual

    -help
        print concise command overview

        Example

            synapseq -help

    -version
        print version build and platform information

        Example

            synapseq -version

    Hub

    -hub-update
        refresh local Hub index

        Example

            synapseq -hub-update

    -hub-clean
        remove cached Hub data

        Example

            synapseq -hub-clean

    -hub-list
        list sequences from local Hub index

        Example

            synapseq -hub-list

    -hub-search WORD
        search local Hub index

        Example

            synapseq -hub-search calm

    -hub-info NAME
        print Hub sequence metadata

        Example

            synapseq -hub-info calm-state

    -hub-download NAME [DIR]
        download sequence and dependencies
        DIR optional

        Example

            synapseq -hub-download calm-state downloads

    -hub-get NAME [OUTPUT]
        download and render sequence
        OUTPUT optional

        Example

            synapseq -hub-get calm-state calm-state.wav

    External tools

    -ffmpeg-path PATH
        use specific ffmpeg executable

        Example

            synapseq -ffmpeg-path /usr/local/bin/ffmpeg -mp3 session.spsq

    -ffplay-path PATH
        use specific ffplay executable

        Example

            synapseq -ffplay-path /usr/local/bin/ffplay -play session.spsq

    Windows

    -install-file-association
        associate .spsq files with SynapSeq

        Example

            synapseq -install-file-association

    -uninstall-file-association
        remove .spsq file association

        Example

            synapseq -uninstall-file-association

SEQUENCE FILE
    File type

        plain-text .spsq
        parsed top to bottom

    Order

        sequence options
        presets
        timeline

    Top level

        no indentation
        sequence options must start in column 1
        preset declarations must start in column 1
        timeline entries must start in column 1

    Preset body

        exactly two leading spaces required
        track definitions use this indentation
        track overrides use this indentation

    Comments

    # comment
        ignored

    ## comment
        stored and printed unless -quiet is set

    Rules
        full line only
        inline comment not permitted

    Sequence Options

    Sequence options configure the whole file and must appear before the first
    preset or timeline entry.

        Rules

            valid only before first preset
            valid only before first timeline entry

        Commands

        @samplerate NUMBER
            set output sample rate
            default 44100

            Example

                @samplerate 48000

        @volume NUMBER
            set master volume
            range 0 to 100
            default 100

            Example

                @volume 80

        @ambiance NAME PATH_OR_URL
            register a named ambiance source for later use in presets

            Example

                @ambiance rain audio/rain

        @extends PATH_OR_URL
            load modular .spsc content before the main file is parsed

            Example

                @extends library/common

        Local paths

            relative only
            forward slashes only
            no extension
            ambiance resolves to .wav
            extends resolves to .spsc
            absolute path not permitted
            parent traversal not permitted
            Windows drive prefix not permitted
            stdin not permitted

        Extended files

            type .spsc
            allowed: options presets tracks track overrides
            not permitted: timeline entries nested @extends

    Presets

    Presets define reusable track groups. Declare them at the top level and
    place track lines inside the body with exactly two leading spaces.

        Identifiers

            first character must be a letter
            remaining characters may be letters digits underscores dashes
            maximum length 20
            preset references are case-insensitive
            preset names normalize to lowercase

        Preset declarations

            NAME
            NAME as template
            NAME from TEMPLATE_NAME

        Preset example

            focus
              noise pink amplitude 30
              tone 180 binaural 10 amplitude 18

        Template rules

            template preset cannot appear on timeline

        Inheritance rules

            inherited preset cannot define new track lines
            inherited preset may contain track overrides only

        Inherited preset example

            base as template
              noise pink smooth 20 amplitude 30
              tone 220 binaural 10 amplitude 18

            focus from base
              track 1 smooth 35
              track 1 amplitude +5
              track 2 tone 180
              track 2 binaural -2
              track 2 amplitude +4

        Track syntax

            tone CARRIER amplitude LEVEL
            tone CARRIER binaural|monaural|isochronic BEAT amplitude LEVEL
            noise white|pink|brown [smooth VALUE] amplitude LEVEL
            ambiance NAME amplitude LEVEL

        Track rules

            amplitude required
            carrier positive only
            beat positive only
            smooth 0 to 100 only
            waveform prefix allowed on tone track
            waveform prefix allowed on ambiance track
            effect TYPE VALUE intensity PERCENT appears before amplitude

        Waveforms

            waveform VALUE

            VALUE

                sine square triangle sawtooth

        Effects

            effect TYPE VALUE intensity PERCENT

            TYPE

                pan modulation doppler

            Allowed

                tone
                    pan modulation doppler

                noise
                    pan modulation

                ambiance
                    pan modulation

        Track overrides

            Scope

                inherited presets only

            Index

                1-based

            Syntax

                track N tone VALUE
                track N binaural|monaural|isochronic VALUE
                track N smooth VALUE
                track N pan|modulation|doppler VALUE
                track N intensity VALUE
                track N amplitude VALUE

            Rules

                VALUE may be absolute or signed relative delta
                +VALUE adds to inherited value
                -VALUE subtracts from inherited value
                matching existing effect type only

    Timeline

    Timeline entries schedule presets over time. Each entry starts with a
    timestamp and may include a transition keyword.

        Syntax

            HH:MM:SS PRESET_NAME [TRANSITION]

        Time

            HH:MM:SS required

        Rules

            top-level lines only
            first entry zero time
            strictly increasing
            at least two periods
            non-template preset only

        TRANSITION

            steady ease-in ease-out smooth

        Timeline examples

            00:00:00 silence
            00:00:30 focus ease-in
            00:10:00 focus-deep smooth
            00:20:00 silence ease-out

COMPATIBILITY
    Scope

        checked between consecutive timeline entries
        checked per channel
        channels match by track declaration order

    Required matches

    track kind
        tone only with tone
        noise only with noise
        ambiance only with ambiance

    beat mode
        binaural only with binaural
        monaural only with monaural
        isochronic only with isochronic

    noise color
        white only with white
        pink only with pink
        brown only with brown

    effect type
        pan only with pan
        modulation only with modulation
        doppler only with doppler

    ambiance source
        same source only

    Not permitted

        active track to off
        off to active track
        direct switch between incompatible track kinds
        direct switch between incompatible effect types
        direct switch between different ambiance sources

    Allowed

        waveform change between otherwise compatible tone tracks
        waveform change between otherwise compatible ambiance tracks

    Bridge

        use built-in silence preset between incompatible states

COMMON ERRORS
    Option after preset or timeline
        move all @options to file start

    Indented top-level line
        remove indentation from preset declarations and timeline entries

    Wrong preset indentation
        use exactly two leading spaces for preset body lines

    Inline comment
        move comment to its own line

    Template on timeline
        replace template with concrete preset

    First timeline entry not zero time
        set first entry to zero time

    Non-increasing timeline
        reorder entries into strictly increasing time order

    Single period only
        add a second timeline entry

    New track in inherited preset
        remove track definition
        use track override only

    Incompatible direct transition
        insert silence between presets

    Invalid local path
        use relative path
        use forward slashes
        omit extension

NOTES
    silence
        built-in preset
        always available

    Channels
        assigned by track declaration order

    Reuse
        @extends loads reusable .spsc content
