# SynapSeq Design System

This folder exists to document and preserve SynapSeq's visual identity.

Even though this is a CLI-centered project, the repository already contains real visual surfaces, such as the HTML preview and the WASM interface. The purpose of this directory is to ensure those interfaces share the same visual language and communicate the same product feeling.

## Purpose

The SynapSeq design system exists to:

- define the project's visual identity explicitly
- align light mode and dark mode under the same aesthetic direction
- prevent divergence between preview, WASM, and future HTML interfaces
- document tokens, surfaces, typography, components, and visual behavior
- keep implementation consistent with design intent

## What This Identity Should Convey

SynapSeq should not feel like only a technical tool or only a contemplative interface. Its visual identity needs to support both sides at the same time:

- calm and immersion
- technical precision
- long-session readability
- sophistication without coldness
- atmosphere without decorative excess

In practical terms, that means a warm, soft, analytical visual language with translucent surfaces, controlled contrast, generous rounding, clean typography, and restrained visual feedback.

## How To Use It

When creating or adjusting visual interfaces in the project:

- consult the specification in [ds-basic.md](/Users/ruan/Dev/synapseq/ds/ds-basic.md) first
- preserve the existing tokens and visual mood
- treat dark mode as a continuation of light mode, not as a separate identity
- prefer evolving the existing patterns instead of introducing new styles unnecessarily

## Scope

This material is not a corporate brand bible or a marketing document. It exists to guide the visual implementation of the SynapSeq interfaces contained in this repository.
