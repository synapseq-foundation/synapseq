## Context

SynapSeq is a CLI tool with multiple flags defined in `internal/cli` package via the `CLIOptions` struct. Currently, users must memorize or check help to discover available flags. Shell completion will improve UX by enabling tab-completion in bash and zsh shells.

## Goals / Non-Goals

**Goals:**
- Provide bash and zsh completion scripts that users can source into their shell config
- Expose a machine-readable flag list via `-completion-args` for use by completion scripts
- Store flag metadata (name + description) in a centralized `map[string]string` in `completion.go`
- Completion output must prepend `-` to flags for correct CLI parsing

**Non-Goals:**
- Interactive shell completion (beyond what bash/zsh native completion provides)
- Completion for positional arguments or file paths
- Windows PowerShell completion support

## Decisions

1. **Centralized flag metadata map** — All completable flags and their descriptions (≤50 chars, no colon) are stored in a `map[string]string` in `completion.go`. This avoids duplicating descriptions across help text and completion logic.

2. **`-completion-args` as data source** — The completion scripts will call `synapseq -completion-args` to obtain the flag list dynamically, rather than hardcoding flags in shell scripts. This ensures completions stay in sync with the binary.

3. **Bash/zsh scripts embedded as strings** — The completion functions are generated by Go code and printed to stdout, allowing users to redirect output to their rc files. This avoids distributing separate script files.

4. **Description constraint** — Descriptions for `-completion-args` output are truncated to 50 characters and validated to exclude the `:` character, ensuring the `{param}:{desc}` format is unambiguous.

## Risks / Trade-offs

- [Shell compatibility] → Scripts target bash 4+ and zsh 5+. Minimal compatibility checks included.
- [Description quality] → 50-char limit may require careful wording; mitigation is to review all flag descriptions for clarity.
- [Dynamic completion overhead] → Calling `synapseq -completion-args` on each tab press is lightweight (no audio rendering), acceptable trade-off for accuracy.
